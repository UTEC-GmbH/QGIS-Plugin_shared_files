
Notes:

  * Create a symbolic link

    Open Command Prompt as an Administrator.
    Create the symbolic link using the "mklink /D" command. 
    The format is: mklink /D <Link_Path> <Target_Path>

    mklink /D "C:\Users\**USER_NAME**\AppData\Roaming\QGIS\QGIS**X**\profiles\default\python\plugins\**PLUGIN_NAME**_dev" "C:\**DEV_DIRECTORY**\**PLUGIN_NAME**"


  * Translation Workflow
    
    Compile the Translation by running compile.bat found in the root directory in the OSGeo4W Shell. 
      - Open the OSGeo4W Shell (from the Windows start menu)
      - `cd **PATH_TO_PLUGIN**`
      - `compile.bat`

    This finds all the strings to translat in the code and 
    opens linguist so the translations can be edited. 
    After closing linguist, it will compile the translations.

    If linguist doesn't open or lrelease is not installed, 
    go to OSGeo4W setup and install "qt6-devel" (in the "Libs" section)


  * Publishing New Versions

    1. Update the `version` number and changelog in `metadata.txt`. 

    2. Run the release script from the OSGeo4W Shell in the plugin directory 
       using the following commands:
       
      `cd **PATH_TO_PLUGIN**`
      `python release.py`
       
      This script will update `plugins.xml` and package the new .zip file.
      The .zip file and plugins.xml will be saved to the shared repo 
      (see `download_url_base` in metadata.txt).
